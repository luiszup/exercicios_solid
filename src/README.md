# Exercícios Resolvidos: Praticando SOLID

Este repositório contém a resolução de exercícios para praticar os princípios SOLID. Cada exercício foi refatorado para seguir os princípios de design de software, promovendo código mais limpo, modular e fácil de manter.

---

## Exercício 1: Single Responsibility Principle (SRP)
### Enunciado
Refatore a classe `RelatorioFinanceiro` para que ela siga o Princípio da Responsabilidade Única (SRP). Separe as responsabilidades de geração de relatório e envio de e-mail em classes diferentes.

### Status
Resolvido utilizando o princípio SRP.

---

## Exercício 2: Open/Closed Principle (OCP)
### Enunciado
Refatore a classe `Calculadora` para que ela siga o Princípio Aberto/Fechado (OCP), permitindo adicionar novas operações sem modificar o código existente.

### Status
Resolvido utilizando o princípio OCP.

---

## Exercício 3: Liskov Substitution Principle (LSP)
### Enunciado
Refatore as classes `Retangulo` e `Quadrado` para que sigam o Princípio da Substituição de Liskov (LSP). Crie uma interface `Forma` e implemente classes separadas para cada forma.

### Status
Resolvido utilizando o princípio LSP.

---

## Exercício 4: Interface Segregation Principle (ISP)
### Enunciado
Refatore a interface `Funcionario` para que ela siga o Princípio da Segregação de Interfaces (ISP). Separe a interface em partes menores.

### Status
Resolvido utilizando o princípio ISP.

---

## Exercício 5: Dependency Inversion Principle (DIP)
### Enunciado
Refatore a classe `Computador` para que ela siga o Princípio da Inversão de Dependência (DIP). Crie uma interface `DispositivoEntrada` e faça com que `Computador` dependa dela.

### Status
Resolvido utilizando o princípio DIP.

---

## Exercício Bônus: Projeto Final com SOLID
### Enunciado
Crie um sistema para gerenciar contas bancárias que siga os princípios SOLID. O sistema deve permitir criar diferentes tipos de contas, calcular taxas específicas e enviar notificações.

### Status
Resolvido utilizando todos os princípios SOLID.
